→ API Layer

FastAPI handles all incoming HTTP requests with automatic OpenAPI documentation, type validation, and async support. Routes are modular — chat, contact, and projects each have their own router.

→ Validation

Pydantic models validate every request and response. This ensures type safety, prevents bad data from reaching the database, and provides clear error messages to the client automatically.

→ Database

SQLite provides zero-config, serverless data storage. The schema includes tables for chatbot responses, contact messages, projects, and chat history — all managed through clean Python database utilities.

→ Chatbot Engine

The chatbot uses keyword-based matching against a database of responses. User messages are tokenized, matched with priority scoring, and the best response is returned. All conversations are logged for analytics.
